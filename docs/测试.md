## 重点观察

- 最新一批正式测试主要集中在 2026-07-14，覆盖 C/C++、Java、PHP、Python、JavaScript/TypeScript、Go 等项目。
- C/C++ 项目中，`sqlite`、`OpenVPN`、`json-c` 已出现构建记录和 `micro_proof`，说明动态验证链路能进入局部证明阶段；但大多数还没有达到真实 `verified` 或 `harness_reproduced`。
- `sqlite` 的 native build 已成功，但全部 finding 都是 `partial_dynamic_proof / micro_proof`，而且之前人工复核显示部分 PoC 存在 synthetic stub 问题，因此该报告更适合作为验证策略改进样例。
- Web/脚本类项目中，`django`、`axios`、`lodash`、`NodeGoat`、`fastapi` 等多数 finding 仍为 `static_only`，说明当前动态验证对这些语言和框架的覆盖还不充分。
- `maccms10` 最新报告已压缩过超长触发链路内容，报告体积从异常膨胀恢复到约 1.26MB JSON；但验证状态仍以 `uncertain` 为主。
- `DVWA` 最新 deep 报告只有 1 个依赖类 finding，反而比旧 standard 报告少很多；这说明不同模式/策略对 Web 漏洞样例的召回差异较大，后续需要单独复核 mining 策略。
